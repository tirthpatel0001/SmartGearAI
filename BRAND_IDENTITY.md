**SmartGear AI** - Intelligent Manufacturing Intelligence Platform for Gear Manufacturing Companies. Professional, industrial, technically advanced, and trustworthy.

### Brand Name & Tagline

- **Precision** - Accurate pricing and quality detection
- **Efficiency** - Optimized production and cost management
- **Reliability** - Predictive maintenance that prevents downtime

## 🎨 Design Theme: Industrial Modern

### Visual Style
- **Aesthetic**: Industrial, corporate, sophisticated
- **Complexity**: Clean and uncluttered
- **Mood**: Professional, trustworthy, technical
- **Target Users**: Factory managers, engineers, quality controllers

### Design Characteristics
- Dark steel blue for authority and stability
- Orange accents for energy and call-to-action
- Clean white cards for content clarity
- Professional sans-serif typography
- Subtle shadows for depth without gaudiness
- Smooth transitions and hover effects

---

## 🎭 Color Psychology & Usage

### Steel Blue (#1E3A5F / #2C5282)
- **Meaning**: Trust, stability, professionalism, industrial strength
- **Usage**: Headers, navigation, primary sections
- **Psychology**: Conveys precision and engineering expertise

### Orange (#D97706)
- **Meaning**: Energy, action, importance, manufacturing
- **Usage**: Accents, buttons, highlights, important metrics
- **Psychology**: Draws attention to key actions and CTAs

### Green (#059669)
- **Meaning**: Success, safety, positive status
- **Usage**: Success confirmations, positive alerts
- **Psychology**: Users recognize immediately as "good"

### Red (#DC2626)
- **Meaning**: Alert, warning, defect, error
- **Usage**: Error messages, defect detection, warnings
- **Psychology**: Immediate recognition of issues

### White (#FFFFFF)
- **Meaning**: Clarity, simplicity, sophistication
- **Usage**: Card backgrounds, text areas, breathing room
- **Psychology**: Reduces cognitive load, improves readability

### Gray (#6B7280)
- **Meaning**: Neutrality, secondary information
- **Usage**: Supporting text, placeholders, disabled states
- **Psychology**: De-emphasizes without losing readability

## 💡 Key Design Elements

### 1. Hero Headers
```
Large, gradient background (dark blue to steel)
Orange top border for accent
White text with clear hierarchy
Descriptive subtitle
Icon representation (⚙️ for manufacturing)
```
**Purpose**: Immediately establishes page context and authority

### 2. Card System
```
White background with subtle shadows
Orange left border (3-4px)
Rounded corners (8px)
Hover elevation effect
Clear padding and whitespace
```
**Purpose**: Groups related content clearly, provides visual hierarchy

### 3. Button System
```
Orange gradient background
White text, bold weight
Rounded corners (6px)
Hover: lifting animation + gradient deepening
Touch-friendly sizing (44px+ minimum)
```
**Purpose**: Clear, actionable, draws user attention

### 4. Metric Cards
```
White background with orange top border
Monospaced font for numbers
Large, bold metric values
Supporting label text
Centered alignment
```
**Purpose**: Quick KPI comprehension, professional appearance

### 5. Status Indicators
```
Success: Green gradient background
Warning: Orange/amber styling
Error: Red gradient background
Info: Steel blue styling
All with icon and descriptive text
```
**Purpose**: Immediate status clarity without reading

## 📊 Layout & Spacing

### Responsive Breakpoints
- **Desktop**: 1920px+ (3-column layouts)
- **Laptop**: 1280-1920px (2-3 column layouts)
- **Tablet**: 768-1280px (2-column layouts)
- **Mobile**: <768px (1-column stacked layout)

### Standard Spacing Increments
- `8px` - Subtle gaps, element spacing
- `16px` - Standard padding, margins
- `24px` - Content padding, section spacing
- `32px` - Header padding, major sections
- `48px` - Page-level spacing

### Touch-Friendly Targets
- Minimum height/width: `44px` for clickable elements
- Padding around small buttons: `8px`
- Spacing between buttons: `12px+`

## 🔤 Typography Hierarchy

### Level 1: Page Title (H1)
```
Font: Inter, 2.2em, Bold (700)
Color: White on dark background or Dark on white
Purpose: Main page heading, establishes context
Example: "Predictive Maintenance & Diagnosis"
```

### Level 2: Section Header (H2)
```
Font: Inter, 1.4em, Bold (700)
Color: Dark blue (#1E3A5F)
Purpose: Section divisions within page
Example: "Configuration Settings"
```

### Level 3: Card Title (H3)
```
Font: Inter, 1.3em, Semi-bold (600)
Color: Dark blue (#1E3A5F)
Purpose: Card and container titles
Example: "Material Rates"
```

### Body Text
```
Font: Inter, 1em, Regular (400)
Color: Dark gray (#111827)
Line-height: 1.5
Purpose: Primary content, descriptions
```

### Supporting Text
```
Font: Inter, 0.9em, Regular (400)
Color: Medium gray (#6B7280)
Purpose: Secondary information, labels
```

### Label/Badge Text
```
Font: Inter, 0.85em, Semi-bold (600)
Color: Varies by context
Text-transform: Uppercase with letter-spacing
Purpose: Form labels, badges, tags
```

### Monospace (Metrics & Code)
```
Font: Roboto Mono, appropriate size, Regular/Bold
Color: Orange (#D97706) for values
Purpose: Numeric display, price formatting, code
```

## 🌟 Visual States & Interactions

### Button States
1. **Default**: Orange gradient, shadow
2. **Hover**: Deeper orange, lifted up 2px, enhanced shadow
3. **Active**: Even darker, pressed appearance
4. **Disabled**: Gray with reduced opacity
5. **Focus**: Clear outline for accessibility

### Card States
1. **Default**: White background, 0 4px shadow
2. **Hover**: Lifted up 2px, 0 10px 15px shadow
3. **Focused**: Blue outline for keyboard navigation
4. **Disabled**: Reduced opacity, grayed out

### Form Input States
1. **Normal**: White background, gray border (1px)
2. **Focused**: Blue outline, enhanced shadow
3. **Valid**: Green left border indicator
4. **Invalid**: Red left border, error message
5. **Disabled**: Gray background, no interaction

## 🎬 Animations & Transitions

### Standard Transition Properties
```css
transition: all 0.3s ease;
```

### Common Animations
1. **Hover Lift**: `transform: translateY(-2px);`
2. **Scale**: `transform: scale(1.05);`
3. **Shadow Enhance**: Shadow value increase
4. **Color Fade**: Background color transition
5. **Pulse**: Subtle attention animation (rarely used)

### Animation Timing
- **Quick Feedback**: 0.15s - 0.2s (button clicks)
- **Element Transitions**: 0.3s (hover effects)
- **Page Transitions**: 0.5s (navigation)
- **Large Animations**: 0.8s - 1s (loading states)

## 📐 Grid & Alignment

### Column System
- **Desktop (3 cols)**: Each column 30% width + gutters
- **Laptop (2 cols)**: Each column 45% width + gutters
- **Tablet (2 cols)**: Each column 48% width + gutters
- **Mobile (1 col)**: 100% width, full padding

### Alignment Principles
- Center title text for emphasis
- Left-align body content for readability
- Center numeric metrics (KPIs)
- Balance white space on all sides

## ♿ Accessibility Features

### Color Contrast Compliance
- **WCAG AA Standard**: Minimum 4.5:1 for text
- **Primary Colors Meet**: 7.8:1 (Orange + White)
- **Our Blues Meet**: 10.2:1+ (Dark Blue + White)

### Text Alternatives
- All images/icons have descriptive labels
- Form inputs clearly associated with labels
- Links have descriptive anchor text
- Heading hierarchy is logical and consistent

### Interactive Elements
- Focus indicators are visible (outline or border)
- Touch targets minimum 44x44 pixels
- Keyboard navigation is fully supported
- No content hidden behind color alone

### Assistive Technology Support
- Proper semantic HTML
- ARIA labels where needed
- Skip-to-content navigation
- Form validation messages are accessible

## 🎓 Usage Guidelines

### When to Use Each Color
- **Dark Blue**: Headers, primary navigation, authority
- **Orange**: Calls-to-action, accents, highlights
- **Green**: Success states, positive confirmations
- **Red**: Errors, warnings, defects
- **Gray**: Secondary text, disabled states
- **White**: Card backgrounds, breathing room

### When to Apply Shadows
- **No Shadow**: Text fields, subtle backgrounds
- **Shadow MD**: Standard cards, buttons
- **Shadow LG**: Elevated sections, hero headers, hovers

### When to Add Borders
- **Top Border**: Card titles, input regions
- **Left Border**: Feature cards, quote blocks
- **Bottom Border**: Section dividers, table headers
- **Outline**: Focus states for keyboard navigation

## 📊 Design Metrics

### Performance Considerations
- **Page Load**: Optimized CSS (minimal file size)

- **Font Sizes**: Use em or rem units for scalability
- **Spacing**: Use pixel units for precision
- **Breakpoints**: Use CSS media queries
- **Icons**: Use web-safe emoji or SVG icons
